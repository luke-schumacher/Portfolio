🏆 ITT Summit Award 2023 - First Place

Customer Digital Twin

Siemens Healthineers • Ongoing

Technical Architecture: Implemented CVAE (Conditional Variational Autoencoder) generating synthetic customer profiles for healthcare technology sales. PyTorch-based model with dual encoders capturing 47 features (demographics + behavioral metrics), 128-dimensional latent space, <200ms inference targeting production deployment on AWS EC2.

Core Challenge: Operated under GDPR/HIPAA compliance while handling severely imbalanced customer segments. Required real-time predictions with full interpretability for non-technical stakeholders—critical constraints for healthcare production systems.

Solutions & Trade-offs: Applied SMOTE oversampling in latent space (not input space) to preserve feature correlations. Added attention mechanism to decoder for feature importance scoring—15% inference slowdown but essential for stakeholder buy-in. Quantized model to INT8 via TensorRT achieving 3x speedup with <2% accuracy loss.

Team & Impact: Led ML engineering on 4-person team plus 2 domain experts. Designed architecture, implemented training pipeline, optimized for deployment. Iterative latent space analysis revealed customer clusters invisible in raw data—directly informed strategy.

Defense/Aerospace Relevance: Experience with compliance requirements, model interpretability, and performance optimization under constraints directly applicable to mission-critical systems requiring auditability, reliability, and real-time response. Weekly stakeholder demos prevented late-stage requirement drift—lesson in managing complex projects with non-technical decision-makers.

Master's Thesis • Ongoing

Agentic Infrastructure Co-Pilot

Multi-Agent System for Fault Diagnosis

System Architecture: Distributed multi-agent system combining RAG (Retrieval-Augmented Generation) with Neo4j knowledge graphs for autonomous infrastructure fault diagnosis. Three specialized agents—Governance (Telekom policies), Hardware (Siemens equipment manuals), Telemetry (Illigo event logs)—coordinate through message queue (RabbitMQ) orchestrated by FastAPI microservices. RAG pipeline uses ChromaDB + GPT-4 for grounded reasoning.

Technical Challenges: Integrated heterogeneous data sources (PDFs, CSV, JSON) with different update frequencies. Agent coordination critical—preventing conflicting diagnoses while maintaining autonomy. System must scale to 10,000+ devices with real-time telemetry. No ground-truth labels for multi-domain faults complicated evaluation.

Solutions: Built unified entity resolution layer with custom parsers (added 200ms latency but ensured consistency). Implemented Dempster-Shafer voting mechanism where agents propose diagnoses with confidence scores—coordinator aggregates probabilistically. Introduced "Mean Time to Innocence" (MTTI) metric—time to correctly identify fault is NOT in given domain, validated with SMEs.

Key Learning: RAG retrieval quality is bottleneck—invested 40% of effort optimizing chunking strategy and embeddings. Agent specialization proved more effective than monolithic models for complex distributed systems.

Defense Applicability: Multi-agent architecture directly maps to C4ISR systems requiring distributed decision-making under uncertainty. Experience with heterogeneous data fusion and autonomous coordination relevant to military operations and logistics optimization.

AISS Project • Completed

Prompt Academy

Agent-Based Educational Platform

Architecture: Interactive learning platform teaching prompt engineering through game mechanics. Vanilla JavaScript + HTML5 Canvas frontend integrated with Groq SDK (Llama-3.1-70B). Five distinct character agents with specialized system prompts and personalities. Client-side state management with localStorage persistence, deployed on Netlify with edge functions for secure API key management.

Core Challenges: Defending against prompt injection—players attempting to "jailbreak" agents and reveal system prompts. LLM non-determinism made progression unpredictable. API latency (2-3 seconds) disrupted game flow. Budget constraint of $50/month for educational deployment serving 200+ users.

Robustness Design: Implemented multi-layer prompt defense: system prompt reinforcement, meta-prompting for jailbreak detection, and fallback responses. Added lightweight response classifier validating agent outputs before display—retry on failures reduced variance by 60%. Trade-off: slightly reduced spontaneity but dramatically improved reliability.

Cost Optimization: Response caching for common interactions plus rate limiting kept deployment under budget while maintaining responsiveness.

Team Role: Technical lead in 3-person student team. Designed agent architecture, implemented LLM integration, built game mechanics.

Learnings & Defense Relevance: Concise, directive prompts outperformed verbose ones—clarity beats length. User testing revealed creative exploits highlighting importance of adversarial thinking in AI system design. Experience hardening LLM agents against malicious inputs applicable to secure chatbots and AI assistants in sensitive environments. System prompt engineering and defensive design critical for trustworthy autonomous systems.

Production System • Siemens Healthineers

MRI Scheduling AI

Transformer-Based Predictive Model

Technical Stack: Custom Transformer architecture with 6 encoder layers predicting MRI machine utilization patterns. Processes 30-day rolling window of historical usage with 15 engineered features (time-of-day, weekday, patient priority, procedure type). Outputs 7-day usage probability distribution per machine. TensorFlow 2.x deployed via TF Serving on Kubernetes with <50ms inference latency requirement.

Data & Performance Constraints: Missing data during maintenance windows with irregular sampling created data quality issues. Multi-scale temporal patterns (hourly, daily, weekly cycles) required careful architecture design. Production threshold: >85% accuracy to impact operations; real-time scheduling decisions demanded <50ms latency.

Solutions: Implemented time-series specific imputation—linear interpolation for short gaps, seasonal decomposition for long gaps preserved temporal structure. Added multi-scale positional encoding capturing both short-term (hourly) and long-term (weekly) patterns—30% more parameters but 12% accuracy gain. Applied knowledge distillation reducing model size 4x while maintaining accuracy within 1% of original, meeting latency requirements on standard hardware.

Team & Results: ML engineer on 5-person team with 2 radiologist domain experts. Designed architecture, built training pipeline, conducted ablation studies, deployed to production. Achieved 89% accuracy enabling 15% improvement in resource utilization.

Key insight & Defense Impact: Close collaboration with domain experts revealed critical patterns (emergency case handling) that dramatically improved model—stakeholder engagement essential. Transformers excel at long-range dependencies but require careful positional encoding for time-series vs NLP. Experience with logistics optimization and resource scheduling directly applicable to defense operations, supply chain management, and mission planning under operational constraints.

ESG Reporting Dashboard

Automated Sentiment Analysis Platform

System Overview: Automated ESG sentiment analysis processing GDELT Global Events Database (50,000+ daily news articles). NLP pipeline: spaCy entity recognition + VADER sentiment scoring. Node2Vec graph embeddings on company mention co-occurrence network. Interactive Streamlit dashboard with NetworkX relationship mapping. Built in 48-hour hackathon achieving dual awards.

Scale & Noise Challenges: Processing 50,000+ articles daily within tight timeline. High noise from irrelevant mentions and entity extraction false positives. Generic sentiment models performed poorly on domain-specific ESG language ("greenwashing," "emissions"). Dashboard required real-time updates.

Pragmatic Solutions: Built ESG keyword list (200+ terms) + company whitelist reducing noise 80%—processed 10,000 relevant articles daily. Fine-tuned VADER lexicon with ESG-specific terms improving sentiment F1 score from 0.62 to 0.78. Node2Vec embeddings revealed implicit relationships (supply chain partners) enabling "related companies" feature for competitive analysis.

Team Role: ML engineer in 4-person hackathon team. Built NLP pipeline, implemented graph embeddings, integrated with dashboard.

Hackathon Lessons: Time pressure forced pragmatic choices—simple models with strong features beat complex models with poor data. Graph-based representations powerful for capturing implicit relationships in unstructured text.

Intelligence Applications: Large-scale text processing and automated relationship extraction directly applicable to OSINT (Open-Source Intelligence), information operations analysis, and threat assessment. Experience with rapid prototyping under constraints and noisy real-world data mirrors intelligence analysis workflows where perfect data rarely exists and decisions require imperfect information synthesis.
